Continue building the Backstage OS app from Phase 1. The app is a modern, elegant, and minimal tool designed for professional stage managers, working either as freelancers or full-time employees at theaters.

In this Phase 2 build, focus on core **workflow tools** used daily in stage management. These features must be flexible, responsive on web, iPhone, and iPad, and designed with performance, user experience, and future scalability in mind.

Expand the app to include the following features:

1. **Script Editor**
   - Allow users to paste or import a full script
   - Automatically paginate based on page breaks or manually inserted breaks
   - Allow manual adjustment of pagination
   - Break scripts into acts, scenes, and pages
   - Include a cue-building tool: users can insert cues (Lighting, Sound, Video, Automation, etc.) at specific locations
   - Cue display should layer visually (e.g., color-coded, toggled by category)
   - Script should be editable and savable per version (for evolving drafts)

2. **Props Tracker**
   - Users can create a props list per project or show
   - Track items by scene and/or character
   - Fields should include: Item Name, Description, Scene, Location, Status (Needed, Pulled, In Rehearsal, In Performance), Notes
   - Attach photos or reference images to each prop
   - Optional: add quantity and sourcing notes

3. **Costume Tracker**
   - Costume list organized by character
   - Allow users to assign costumes to specific scenes
   - Track notes like quick changes, repairs, or special handling
   - Fields: Character, Costume Piece, Scene, Notes, Status, Photo
   - Ability to print/export quick-change tracking for backstage use

4. **Template Customization System**
   - Allow users to edit or create new report templates
   - Editable sections: note categories, distribution lists, headings
   - Users can save custom templates per project or globally to reuse later
   - Templates can be selected when creating a new report

Design requirements:
- Maintain a clean, modern UI with responsive layout
- All features must work seamlessly on desktop and mobile
- Prioritize clarity, speed, and minimal clicks for common tasks

Keep all existing functionality from Phase 1 intact. Use scalable architecture and modular code that supports future features like chat, billing, and real-time collaboration.

The experience should make stage management faster, smarter, and easier—while staying adaptable for both freelancers and full-time SMs.
