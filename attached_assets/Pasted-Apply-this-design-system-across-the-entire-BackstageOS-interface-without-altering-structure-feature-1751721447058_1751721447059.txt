Apply this design system across the entire BackstageOS interface without altering structure, features, or functionality. This is a global design update focused solely on unifying the visual style and interaction design to match the clean, minimal, and user-friendly aesthetic of Notion.

⸻

Design Goals
	•	Preserve all existing components, logic, layout structure, and feature flows
	•	Do not change routing, content hierarchy, or database-bound logic
	•	Focus on unifying appearance, spacing, and component behavior

⸻

UI Foundation (Match Notion’s Visual Language)
	•	Typography: Use Inter or system-ui for all body text and headers
	•	Light-weight line height (1.4), generous vertical spacing
	•	Headings (H1–H4) should feel readable, not oversized
	•	Color Scheme: Neutral grayscale base
	•	Text: #37352f for body, #2f3437 for headers
	•	Background: Pure white #ffffff with soft gray section dividers #ebeced
	•	Accent (optional): Muted blue #3b82f6 for active states and links
	•	UI Padding & Spacing:
	•	16–24px horizontal padding for primary containers
	•	12–16px vertical spacing between grouped elements
	•	Consistent 8px baseline spacing for internal form fields and content blocks

⸻

Component Styling
	•	Cards & Panels: Use subtle border-radius (6px), no heavy box shadows
	•	Soft border: 1px solid #e1e1e1 or #ddd
	•	Light hover state with #f8f9fa background
	•	Inputs:
	•	Rounded corners, soft focus states
	•	No harsh outlines; use box-shadow or subtle border color change on focus
	•	Buttons:
	•	Primary: solid with muted accent color (#3b82f6), no shadow
	•	Secondary: border-only or text button, matching font weight
	•	Sizing: uniform padding (px-4 py-2), rounded (rounded-md)
	•	Menus & Dropdowns:
	•	Minimal with soft borders and gentle fade-in
	•	Grouped sections with light separators
	•	Rich Text Areas (Script Editor, Reports, etc.):
	•	Use a Notion-style clean white background
	•	Remove all unnecessary outlines and visual clutter
	•	Maintain consistent font, spacing, and markdown-friendly shortcuts

⸻

Optional: Dark Mode
	•	Add support for darkMode toggle using Tailwind’s media or class strategy
	•	Match Notion’s dark UI: #1d1d1f background, #f5f5f5 text, accent blue #4e9af1

⸻

Interactions & Transitions
	•	Subtle fade or scale animations on modals, dropdowns, and tooltip triggers
	•	Cursor pointer on all clickable elements
	•	Focus states must be keyboard-accessible and visible

⸻

Constraints
	•	Do not modify the layout structure, backend logic, or component hierarchy
	•	Do not introduce any third-party UI libraries
	•	Keep the design system modular and compatible with Tailwind + Shadcn/UI
	•	All design updates should be reusable and driven by global Tailwind classes, not inline styles
